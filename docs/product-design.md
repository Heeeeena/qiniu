# 产品设计

## 目标用户

- 独立游戏开发者
- Game Jam 参赛者
- 2D 原型设计团队
- 缺少固定美术资源的小型项目组

## 用户痛点

- 单个素材生成容易，成套素材风格统一较难。
- 游戏开发需要的不只是图片，还包括尺寸、透明背景、Sprite Sheet、metadata 等工程化资产。
- 临时找素材成本高，授权和风格匹配都不稳定。

## 产品定位

GameAssetForge 不是通用图片生成器，而是面向 2D 游戏项目的素材生产工作台。它把素材需求、风格约束、批量生成、预览筛选、工程导出放在同一个流程中。

## MVP 范围

- 五类素材：角色、道具、瓦片、背景、UI。
- 风格包：风格、色板、尺寸、透明背景、一致性种子。
- 风格包管理：保存多套项目风格包，并可一键应用到后续生成。
- 本地生成：稳定输出可演示 PNG。
- 质量验收：生成后检查尺寸、Alpha 通道、metadata、Sprite Sheet 适配和风格一致性。
- 导出：PNG、ZIP、Sprite Sheet、metadata.json、spritesheet.json、engine-import-guide.md。
- 引擎预设：Unity、Godot、Cocos、Tiled、Aseprite。
- 批次命名：通过命名前缀保证文件名、帧名、metadata 一致。
- 历史：本地保存最近生成素材。

## 评审关键词对应

- 生成效率：批量生成、历史复用、一键导出。
- 素材质量：类型、风格、色板、尺寸、透明背景约束。
- 风格一致性：项目风格包、色板和一致性种子贯穿同一批次。
- 融入流程：导出包包含独立 PNG、Sprite Sheet、帧坐标 manifest 和目标引擎导入指南。
